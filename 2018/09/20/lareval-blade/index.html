Laravel 的 Blade 模板支持继承，这意味多个子视图可以共用父视图提供的视图模板。接下来让我们修改之前创建的首页视图文件，来学习下如何使用 Blade 模板的继承。


@extends(‘layouts.default’)
@section(‘content’)

主页


@stop

我们使用了 @extends 并通过传参来继承父视图 layouts/default.blade.php 的视图模板。

使用 @section 和 @stop 代码来填充父视图的 content 区块，所有包含在 @section 和 @stop 中的代码都将被插入到父视图的 content 区块

注意的是，当 @section 传递了第二个参数时，便不需要再通过 @stop 标识来告诉 Laravel 填充区块会在具体哪个位置结束。

Couldn’t find that App?

一开始是提示 403，按照下面的提示，将网上的项目删掉了，想要重新来。 heroku create 没问题，但是rename 或 heroku buildpacks:set heroku/php 时都提示 Couldn’t find that app 。然后发现，命令试图修改的，还是刚刚删除的那个app的名字。怎么切换到当前新建的app呢？ 还有就是，如果新建了好几个app，heroku 可以在它们之间切换吗？在网上搜了一下heroku的命令，没有找到关于切换的命令。

附言 1 · 1个月前
找到答案了：
heroku remote -v //查看远程的源
若远程的源不是你新建的 heroku app 地址。则删除之，重新添加源
删除方法: git remote rm heroku
新增方法： heroku git:remote -a yourapp
